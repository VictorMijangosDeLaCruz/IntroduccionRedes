Datos no estructurados¶

Los datos no estructurados son aquellos datos que no tienen una organizacin a priori y cuya estructura no es inmediatamente obvia. Este tipo de datos representan un mayor reto para ser procesados, pues al no contar con una organizacin y caracterizacin previas, pueden presentar variaciones (de tamao, de longitud, etc.) y suelen ser ms difciles de procesar por un sistema de aprendizaje.

Algunos datos no estructurados comnes son los siguientes:

  1. Imgenes
  2. Texto
  3. Grficas relacionales

Imágenes¶

Las imgenes son representadas como matrices de pxeles. En el caso ms sencillo, un pxel puede ser un slo valor numrico, por lo que la imgen podr representarse en una matriz de tamao $H \times W$, donde $H$ es la altura en pxeles de la imgen, y $W$ su anchura. Un ejemplo lo proporciona el conjunto de datos MNIST, que cuenta con imgenes de dgitos escritos a mano:

Sin embargo, los pxeles no suelen representarse slo con un valor numrico, pues esto dara imgenes slo en escala de grises. Es comn usar el formato RGB (Red-Green-Blue) para representar los pxeles de una imagen. El RGB representa cada pxel como un vector 3-dimensional, donde cada entrada representa la cantidad de color rojo, verde o azul que contiene ese pxel. De tal forma, que se pueden obtener diferentes colores para representar imgenes ms elaboradas.

Asimismo, se pueden representar imgenes en formato RGB-$\alpha$, donde se agrega un valor extra $\alpha$ que representa la transparencia del pxel. En este caso, entonces, cada pxel ser un vector 4-dimensional.

De esta forma, las imgenes se representarn por tensores de rango 3 con tamao $H\times W \times C$, donde $H$ y $W$ son la altura y anchura en pxeles, respectivamente, y $C$ es el nmero de canales (RGB o RGB-$\alpha$).

Flattening¶

El flattening permite transformar una imagen (o cualquier tensor) en un vector. As, un tensor $T$ en $\mathbb{R}^{H \times W \times C}$, puede aplanarse en un vector de tamao $\mathbb{R}^{H \cdot W \cdot C}$. Para esto, podemos usar la opcin de reshape presente tanto en numpy como en PyTorch.

El flattening permite procesar las imgenes con un algoritmo de aprendizaje. Por ejemplo, con un perceptrn multicapa o algn otro mtodo que tome vectores como entrada. Sin embargo, procesar as las imgenes suele perder informacin, pues si la imagen es trasladada o rotada, los vectores resultantes del flattening seran diferentes a pesar de representar el mismo objeto. Datos textuales¶

Otro tipo de dato no estructurado comn son los datos en formato de texto. El caso del texto es ms complejos que las imgenes, pues no se encuentran en un formato tensorial de principio. Se trata de cadenas de smbolos. Una cadena de smbolos puede tener la extensin de una palabra, una oracin o un texto completo. Por tanto, se deben elegir cules son las unidades que se buscan representar. A estas unidades se les llama tokens. En la prctica se suelen usar tokenizadores sofisticados que encuentran sub-palabras (subwords) que son cadenas que idelamente tienen un significado dentro del lenguaje.

Si bien las palabras como espacios entre blancos no es la forma ms eficiente de representar las palabras, es una forma sencilla para observar cmo se pueden representar los datos textuales en formato tensorial. En lo que sigue seguimos los siguientes pasos:

  • Definimos un vocabulario que contenga los tokens (palabras) y que les asigne un valor numrico.
  • Reemplazamos los datos textuales por arreglos de nmeros.
In [7]:
from collections import defaultdict, Counter
import torch
import torch.nn as nn

#Funcion que crea un vocabulario de palabras con un indice numerico
def vocab():
    vocab = defaultdict()
    vocab.default_factory = lambda: len(vocab)
    return vocab    

#Funcion que pasa la cadena de simbolos a una secuencia con indices numericos
def text2index(corpus, vocab):
    for doc in corpus:
        yield torch.tensor([vocab[w] for w in doc.split()+['EOS']], dtype=torch.long)

Para los modelos de aprendizaje resulta ms fcil trabajar con datos numricos, por lo que los datos de texto, los tokens, se sustituyen por ndices numricos en las cadenas en que estos tokens aparecen. Los ndices asignados son arbitrarios, y slo basta llevar un registro, el vocabulario, de qu ndice representa qu palabra.

En el aprendizaje profundo es comn representar las textos mediante encajes o embeddings, que son representaciones vectoriales de cada token en un texto. En PyTorch y Tensorflow ya existen las capas de Embedding que, a partir de los ndices numricos que hemos asignado, representan los vectores.

Aqu vemos el uso de la capa Embedding en PyTorch. Primero definirmos el tamao del vocabulario $n$; es decir, el nmero de tokens que deseamos representar. As como las dimensiones en las que queremos que los tokens se representen. En este caso, para visualizar ms adecuadamente usamos 2 dimensiones.

De esta forma, la capa de embedding toma un dato textual, por ejemplo una oracin, y regresa una matriz donde cada rengln es la representacin vectorial de un token en el texto.

Sin embargo, el uso de las capas de embedding se integra dentro de arquitecturas neuronales ms complejas, orientadas a resolver problemas especficos. De tal forma, que la red neuronal aprende a resolver el problema mientras aprende las representaciones vectoriales de los tokens en el texto. Segn el problema los embeddings resultantes pueden variar. En la prctica se suelen usar embeddings pre-entrenados; es decir, representaciones vectoriales de tokens que son entrenados para tareas generales, de tal forma que pueden adaptarse de manera sencilla a tareas especficas. Una de estas tareas generales es la prediccin del siguiente token, que de hecho es la tarea que presentamos a continuacin.

from tqdm import tqdm

#Arquitectura de la red
out = nn.Sequential(nn.Linear(2, len(voc)+1), nn.Softmax(1))
risk = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(list(emb.parameters()) + list(out.parameters()), lr=0.01)
#Entrenamiento
for t in tqdm(range(0, 100)):
    for x in data:
        y_pred = out(emb(x[:-1]))
        optimizer.zero_grad()
        loss = risk(y_pred, x[1:])
        loss.backward()
        optimizer.step()

Grafos relacionales¶

Las relaciones entre diferentes objetos en un sistema relacional se pueden representar adecuadamente por medio de grafos. Por ejemplo, interaccin en redes sociales, estructuras qumicas, etc. Los grafos son una herramienta poderosa para representar estos datos. Sin embargo, los grafos entran dentro de lo que consideramos datos no estructurados, pues no existe una estandarizacin entre las relaciones que se pueden dar y, por tanto, no podemos decir que estn organizados bajo criterios especficos.

Los grafos pueden ser pesados, mostrando un valor numrico por cada relacin. Dentro del aprendizaje automtico y el aprendizaje profundo, los grafos tienen un papel sumamente importante.

Los grafos pueden representarse tambin por medio de tensores. En particular, todo grafo tiene una matriz de adyacencia asociada, que representa las conexiones de los elementos en los grafos. En el aprendizaje profundo, tmabin es comn que los nodos tengan asociado un vector que los represente. Esto es comn en redes neuronales para grafos, que adems de procesar estas representaciones vectoriales de los nodos, utilizan la informacin de la amtriz de adyacencia para resolver tareas.
